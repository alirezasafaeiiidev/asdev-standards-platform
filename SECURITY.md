# Security Policy

## Supported Scope

This repository contains governance standards, templates, and automation scripts.
Security reports are accepted for:
- Script supply-chain risks
- Credential leakage risks
- Workflow permission misconfigurations
- Unsafe automation behavior affecting repositories

## Reporting a Vulnerability

Please report vulnerabilities privately by opening a GitHub Security Advisory for this repository.
If advisory access is unavailable, open a private report via GitHub contact channel for the repository owner.

Include:
- Affected file/path and commit SHA (if known)
- Reproduction steps
- Impact assessment
- Suggested mitigation (optional)

## Response SLA

- Initial triage acknowledgment: within 72 hours
- Mitigation plan or status update: within 7 days
- Fix timeline depends on severity and blast radius

## Disclosure Policy

Please do not disclose vulnerabilities publicly until a fix or mitigation is available.
